二、新版生命周期

2.1 改动

在原来的基础上遗弃了三个钩子函数，新增了两个钩子函数
如果同时使用遗弃的和新增的钩子会报错，如下图:
componentWillMount,组件将要挂载前触发以后版本要加上UNSAFE_连接，以后可能会不用了
componentWillReceiveProps传递的props更新前触发,同上
componentWillUpdate组件将要更新前触发，同上上
getDerivedStateFromProps,后续有详细描述
getSnapshotBeforeUpdate在componentDidUpdate前触发

2.2 getDerivedStateFromProps

接收一个参数props,它一定要返回一个state状态或者null,然后后面就不能改变组件身上的状态了
只能在state的状态是由props决定的时候才使用，后期只能通过props改变才能改变state

2.3 getSnapshotBeforeUpdate

接收两个参数，必须要返回一个值,两个参数是更新前的props(prevProps)和更新前的state(prevState)
然后后面的生命周期函数能接收到三个参数，分别是prevState、prevProps、返回值
