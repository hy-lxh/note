早安英文android

公司这边的app前期余老师已经搭好了，android方面的打包只需要修改几个东西就好了

1、版本

pubspec.yaml文件里面的version,比如现在是0.1.83+183,那下一次发布就是0.1.84+184
发布完成后将apk包放在cdn.letabc.com/assets/zaxt-app/apk/release/目录下
命名:zaoan-平台-版本.apk

2、平台

修改平台是在/lib/config.dart里面,通过枚举Flavor,华为就修改为Flavor.HUAWEI
其中不一样的是ios端和官网都是Flavor.OFFICIAL

ios发布

正常部分

首先进入(app dir)/ios目录下,点击Runner.xcodeproj或者Runner.xcworkspace文件进入xcode
选中Runner -> General,修改version和build,这个其实就是android之前说的版本
选择⑤部分为any device
选择上方导航栏的product①,点击Artchive②等待编译,如果是灰色的,那就是上一步⑤的问题,没有选择any device
之后弹出build success信息，那么ios已经打包完成了，接下来就是上传了

ios上传测试

打包完成会弹出一个弹窗,选中最近的打包文件,这个xcode会自动选中的，那我们直接点击Distribute App②
下一步选中App Store Connect,这个也是默认选中的，我们点next②下一步就好了
再下一个就选中Upload①,点next②
next
再下一步就要选择另一个Manually manage signing,这个xcode选择相反的了,选择①,点击②
下一个就是涉及一个名叫pp证书这么个东西,这个我已经申请过了,但是这个东西含有时效性，会过期,所以后续可能还需要再次申请，同时前面还有一个证书，到时候后面都会讲清楚如何去申请
如果证书没过期就直接用现成的就好了,在Downloads根目录,这个可以直接点下拉框，因为之前已经选择过了，有记录
选完了,点next,等待校验小会,最后点Upload,等待上传完成就好了
报错了，这个错也是个案例,就是TestFlight已经有这个版本了，我们回去重新提高一个版本再重头开始就好了,手动[苦笑]
出现下图的标志就代表成功上传了
完成到这，测试那边还不能下载我们的app，最后我们要前往App Store那边
网址:App Store
点击app再选择TestFlight就能看到我们上传过来的app了
需要手动点击管理③,选择是点下一步直到完成测试部分就完成了

ios app上线

在相同的网页上选择App Store①，再点IOS App旁边的＋②
输入最新的版本号,点击创建，然后就多出一个准备提交的app版本
输入该版本做的一些东西
选择发布构建版本
修改联系人信息
提交审核

非正常版本

所谓非正常其实就是一个存新的项目拉到mac本里面，如何运行跑起来，就是之前的ios的bug
首先进入xcode，选择Runner -> Flutter -> Generated.xcconfig,修改这里面的路径
修改完成是这样的
进入项目根目录终端,执行flutter pub get
这个错没关系，就是一个警告提醒
进入app dir/ios目录，执行pod install
从别的项目里面copy一个名Runner.entitlements文件,copy到app dir/ios/Runner目录下，如果没有就新建一个flutter应用，这个是没关系的，都是一样的，但是新建一个flutter项目里面是没有这个文件的，必要要开启推送或者其它东西才会生成，这时你可能会问为什么不在自己的项目里面开启推送就能生成，何必这么麻烦，解答一下，因为自己的项目开启推送不会生成，奇了个怪了

开启推送

首先选中Runner① -> Singing Capabilities② -> +Capability③,找到一个名为Push Notifications④的选项，双击
然后就到app dir/ios/Runner目录下找到Runner.entitlements文件(highlight)copy到项目的相应目录下
文件找到后，回到xcode，选择Product① -> Scheme② -> edit Scheme③
出现相应弹窗,选择Run,把Release改为Debug
将Build Settings -> Build Active Architecture Only -> Debug设置为Yes,最后就可以跑起来了，后面的流程都和前面是一样的了
还有一个东西，就是ios需要一张1024x1024大小的logo图，这个得上传上去，不然上传会报错
这张图片在电脑的Downloads/目录下

传图片

选择Runner① -> Runner② -> Assets.xcassets③ -> AppIcon④ -> 1024 x 1024⑤,将图片拖入位置就直接上传了
最后，都配置完了，因为mac本上传过代码了，所以一些设置和配置已经同步过去了，这块了解一下就好了

申请证书

pp证书

进入ios开发者平台登录
选择Overview① -> Centificates,Identifiers & Profiles②
选择Profiles① -> Profiles +②
选择IOS App Development,点击Continue
选择相应的App ID，点击Continue
选择相应设备，点击Continue
全选所有设备，点击Continue
输入文件名，点击Generate
最后下载下来就好了

Certificates证书

打开电脑的钥匙串访问，点击上面的钥匙串访问 -> 证书助理 -> 从证书颁发机构请求证书
输入邮箱，选择存储到磁盘，点击继续
选择储存位置，点击储存
然后再回到ios开发者平台，选中Certificates,点击上方的＋
选中IOS App Development,点击Continue
选择刚刚在钥匙串里面创建储存的证书，点击Continue
点击Download，证书就好了
最后找到相应的证书双击把它添加到钥匙串里面，如果拒绝了，直接拖进去就好了
